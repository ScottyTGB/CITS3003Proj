Documentation Overview

This documentation has two main parts: high-level documentation and "most important files" (low-level documentation).

The high-level documentation provides an overview of the entire project. It includes an outline of the main render loop function hierarchy, which shows what is involved in rendering each frame, and it includes a list of summaries for every file in the project, to help with understanding what each file is responsible for.

The low-level documentation includes detailed descriptions of the functions within various important files. These files are ones that you will likely have to familiarize yourself with over the course of the project.


Tips to Understand the Project Codebase

Understanding any codebase takes time and effort, but here are some tips to make the process easier:

  • Interact with the project: Spend some time running the base project and experimenting with the different ImGui elements.

  • Refer to the low-level documentation: As you work on the project tasks refer to the low-level documentation for the file you are working on, if it exists. If there is no low-level documentation, find the file in Project File Summaries and you will find a summary of the file's contents and how it fits into the project.

  • Inspect header files for comments: Be sure to look at the header files (.h) of the .cpp files you are working on. These files often contain useful comments that explain the purpose of the class's functions. If there are only a few comments in the header file, identify the implemented class and look for the corresponding header file for more details. For example, in PanningCamera.h you can see from the line "class PanningCamera : public CameraInterface" that PanningCamera implements CameraInterface, so it may be worthwhile to look at CameraInterface.h for comments on what each function in PanningCamera does.

  • Use code search tools: Tools such as grep (Linux) or grepWin (Windows) can simplify the search for specific functions or variables in the codebase. These tools can point you in the right direction with regards to what files you must work on to achieve a desired functionality.

  • Explore the Main Render Loop Function Hierarchy: Take a look at the Main Loop Function Hierarchy to get an overview of how the project renders a frame.


What Exactly is ImGui?

ImGui (Immediate Mode GUI) is a graphical user interface library. It follows the immediate mode paradigm which means the user interface is constructed and rendered every frame, as opposed to the retained mode paradign where the user interface is built and retained from frame to frame.

If you learn best by looking at existing code and adapting it to your needs, I recommend exploring the following functions within the project:

  • In SceneElement.cpp, look at the add_local_transform_imgui_edit_section function.

  • In PanningCamera.cpp, look at the add_imgui_options_section function.

  • In PointLightElement.cpp, look at the add_imgui_edit_section function.

It is unlikely that you will need to use any ImGui functions that are not already used in the project, however if you wish to learn more about ImGui you can read the documentation available here.
