Location in Project

src/rendering/cameras/PanningCamera.cpp


Functionality and Purpose of this File

This file contains all of the functions associated with the panning camera. This includes the constructors, update function, reset function, ImGui elements, save and load functions, and various accessor functions.


Method Documentation


PanningCamera::PanningCamera()

This is the default constructor for the panning camera. It initialises distance, focus_point, pitch, yaw, near, and fov to the corresponding init_ values defined in PanningCamera.h.

Where is this called?

In the base project provided, this constructor is not used anywhere.




PanningCamera::PanningCamera(float distance, glm::vec3 focus_point, float pitch, float yaw, float near, float fov)

This is the constructor used in the base project, which sets the panning camera's initial state and reset state to the specified values.

Where is this called?

This is used in both the open and set_camera_mode functions found in EditorScene.cpp and BasicStaticScene.cpp. open is called whenever we switch to a scene and is responsible for setting up all the default elements of the scene. set_camera_mode is called whenever we are swapping the camera mode between panning and flying.




void PanningCamera::update(const Window& window, float dt, bool controls_enabled)

This function contains the camera logic that is performed every frame. This calculates changes to the camera's focus point, pitch, yaw, and distance based on the movemement of the mouse and what mouse buttons are being pressed. It also re-calculates the view matrix and projection matrix based on these values.

Parameters

const Window& window: The window created upon launching the project.

float dt: The time (in seconds) since the last frame was calculated. This value is set via the update function in WindowManager.cpp, which is called every frame inside the main render loop of main.cpp.

bool controls_enabled: If true, the user can use mouse movement and button presses to change the characteristics of the panning camera, otherwise they are not able to.

Where is this called?

This function is called inside the tick_scene function of SceneManager.cpp, which is in turn called inside the main render loop of main.cpp.




void PanningCamera::reset()

This function sets the panning camera's distance, focus point, pitch, yaw, fov, near, and gamma values to the initial values, which were set when the camera was constructed.

Remarks

You may notice that resetting the panning camera resets it to values that are different to those defined in PanningCamera.h. This is because the initial values used are actually those defined in EditorScene.h or BasicStaticScene.h, depending on what scene option you are currently using (the default interactive scene is the Editor Scene). To understand the reason for this behaviour, look at the set_camera_mode function in either EditorScene.cpp or BasicStaticScene.cpp and you will see that we construct the panning and flying cameras using the init_ values defined in the corresponding header file, instead of those defined in PanningCamera.h or FlyingCamera.h.

Where is this called?

This is only called in the update and add_imgui_options_section functions in PanningCamera.cpp, when the 'r' key is pressed and when the 'Reset (R)' ImGui button is clicked, respectively.




void PanningCamera::add_imgui_options_section(const SceneContext& scene_context)

This function adds the ImGui elements that allow the user to interact with the panning camera's properties via the UI. It is called every frame and allows the user to modify the camera's focus point, distance, pitch, yaw, near plane, fov, and gamma, and also allows the user to reset the camera to it's initial state when it was constructed.

Parameters

const SceneContext& scene_context: An object (defined in SceneContext.h and created in main.cpp) that holds information on the window, window manager, model loader, texture loader, and a bool that indicates if ImGui elements are enabled.

Where is this called?

This is called inside add_imgui_options_section in SceneManager.cpp, which is in turn called inside the main render loop of main.cpp.




CameraProperties PanningCamera::save_properties()

This function returns a CameraProperties struct that consists of the panning camera's position (pulled from the inverse view matrix, look at CameraInterface.h to see what get_position does), yaw, pitch, fov, and gamma.

Returns

CameraProperties: A struct defined in CameraInterface.h, this holds properties common to both the flying and panning camera to allow easier swapping between them.

Where is this called?

This is called in the set_camera_mode functions in both BasicStaticScene.cpp and EditorScene.cpp. In that function, save_properties is used to get the current properties of the panning camera, which are stored and used later as an input for load_properties.




void PanningCamera::load_properties(const CameraProperties& camera_properties)

This function takes a set of camera properties (position, pitch, yaw, fov, and gamma) and uses them to set the corresponding values for the panning camera. Some logic is used to convert from a camera position to a focus point and distance.

Parameters

const CameraProperties& camera_properties: A struct defined in CameraInterface.h, this holds properties common to both the flying and panning camera.

Where is this called?

This is called in the set_camera_mode functions in both BasicStaticScene.cpp and EditorScene.cpp. In that function, load_properties is used to set the properties of the new camera once it has been constructed. The values used for each property are those returned by save_properties.




Various Accessor Methods

These are used to get specific information from the panning camera.

Where are these called?

get_view_matrix, get_projection_matrix, and get_gamma are all used inside use_camera in BaseEntityShader.h. get_inverse_view_matrix is used inside get_position in CameraInterface.h, and get_inverse_projection_matrix is not used anywhere in the base project.
