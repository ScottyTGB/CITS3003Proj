Location in Project

src/rendering/scene/Lights.h


Functionality and Purpose of this File

In this file two structs are declared: PointLight and LightScene. The PointLight struct is a representation of a point light scene element, and the LightScene struct contains a set with all point lights and a helper function to get the nearest point lights up to a specified max_count.


Contents Documentation


struct PointLight

This struct includes the declaration for what parameters we must provide when creating a new point light element, which are the two attributes each point light has - position (a glm::vec3) and colour (a glm::vec4), also declared as member variables in the struct. It also includes a Data struct which is relevant for the point_lights_ubo declaration in BaseLitEntityShader.h and by extension the set_point_lights method in BaseLitEntityShader.cpp.

Remarks

You will notice that the colour attribute inside the PointLight struct is a glm::vec4, but in the Data struct it is a glm::vec3. This is because the 4th element of the point light's colour is used to store a scalar that is applied to the other colour values before it is assigned to the point_lights_ubo. I suggest you look through set_point_lights in BaseLitEntityShader.cpp to see exactly what is going on.




struct LightScene

This struct includes the declaration for an unordered set point_lights as well as a function get_nearest_point_lights. New point lights are added to and removed from the unordered set via insert_light and remove_light in MasterRenderScene.cpp. Functionally, this set is primarily used in get_nearest_point_lights, which is defined in Lights.cpp.

Remarks

get_nearest_point_lights is called in two places of the code - the render function in EntityRenderer.cpp and the render function in AnimatedEntityRenderer.cpp.
